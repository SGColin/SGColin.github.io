Approximate Algorithms (1) : Complexity Classes

I - P and NP

Polynomial Time Algorithm

Algorithm A runs in polynomial time if for every string \(s\) , \(A(s)\) terminates in \(\leq p(|s|)\) "steps".

Where \(|s|\) is the length of \(s\) in memory, and \(p(\cdot)\) is some polynomial function.

所以枚举约数复杂度 \(O(\sqrt n)\) 并不能算多项式算法，因为复杂度是 \(O(\sqrt{2^{|s|}}) = O(2^{\frac{|s|}{2}}),\ |s|=\log_2n\) 。

P (Poly-time Problem)

P = Set of decision problems for which there exists a polynomial time algorithm.

注意讨论的对象是 Decision Problem , 即判断 \(X\) 是否在给定集合 \(S\) 中，回答为 "yes" 或 "no" 。

一个比较著名的 P 问题是素数判断，Agrawal-Kayal-Saxena Algorithm (2002) ，获 2006 年哥德尔奖。

NP (Non-deterministic Poly-time Problems)

NP = set of decision problems for which there exists a certifier \(C(s, t)\) satisfying :

  • \(C(s, t)\) is a poly-time algorithm;

  • Certificate (or witness) \(t\) is of polynomial size: \(|t| \leq p(|s|)\) .

Algorithm \(C(s, t)\) is a certifier for problem \(\mathrm{X}\) if for every string \(s\) :

  • \(s \in X \Leftrightarrow\) there exists a string \(t\) such tht \(C(s, t)=\) yes.

注意只需要在回答 yes 的时候都存在 certificate 符合要求，no 的时候无法提供 certificate，不用要求。

Comparison

P. Decision problems for which there exists a poly-time algorithm.

NP. Decision problems for which there exists a poly-time certifier.

P \(\subseteq\) NP is obvious (use the algorithm as the certifier) . Whether P = NP or not is still a famous open problem.
